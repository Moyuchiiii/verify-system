# cross-verify

## 使い方

```
/cross-verify [クレーム]
```

例:
- `/cross-verify "ChatGPTの月間アクティブユーザーは2024年末時点で3億人"`

## 実行フロー

1. **検索戦略設計**: クレームを検証するための検索アプローチを3方向設計する
   - 一次ソース（公式発表・プレスリリース）
   - 独立メディア（報道機関・業界誌）
   - 学術・研究機関
2. **investigator並列起動**: 3方向それぞれを独立したinvestigatorエージェントで並列実行する
3. **結果統合**: 3つの調査結果を統合し、コンセンサスを判定する
   - 3/3一致: 「強い合意」
   - 2/3一致: 「概ね合意（留保あり）」
   - 1/3以下: 「合意なし・要追加調査」
4. **verifier起動**: 統合した証拠セットに対してACH分析を実行する

## 出力形式

```markdown
# クロス検証レポート

クレーム: [検証対象]
検証日時: YYYY-MM-DD HH:mm

## ソース別結果

| ソース方向 | 結果 | 主要ソース |
|---|---|---|
| 一次ソース | ✅/⚠️/❌/🔍 | [URL] |
| 独立メディア | ✅/⚠️/❌/🔍 | [URL] |
| 学術・研究 | ✅/⚠️/❌/🔍 | [URL] |

## コンセンサス
[強い合意 / 概ね合意 / 合意なし]

## 最終判定
[verifierの判定]
```
